# Licensing & Attribution

Legal compliance information for Arcanus OS (Linux Mint remaster).

## Linux Mint Attribution

Arcanus OS is built upon **Linux Mint 22 Cinnamon**, which is licensed under the GNU General Public License (GPL).

**You must:**
- ✅ Maintain attribution to Linux Mint in all distributions
- ✅ Include original GPL license text
- ✅ Provide source code availability information
- ✅ Clearly state it's a derivative work

**Our approach:**
- Include `etc/issue` file mentioning "Based on Linux Mint 22"
- Keep `/usr/share/doc/linuxmint/` unchanged
- Provide this repo as source code
- License derivative as GPL-2.0

## Arcanus Branding

The Arcanus branding, logo, and visual identity are:
- **Copyright:** MDJ Team / Arcanus Finance
- **Usage:** For demonstration and internal use only
- **License:** Proprietary (not included in GPL)

When distributing Arcanus OS:
- Do not claim Arcanus branding as open-source
- Include copyright notice: "Arcanus is a trademark of MDJ Team"
- Clarify that only Linux Mint core is GPL-licensed

## Compliance Checklist

Before public release:

- [ ] Include `COPYING` file (GNU GPL-2.0 text)
- [ ] Add `CONTRIBUTORS` file crediting Linux Mint team
- [ ] Document in README which components are GPL vs. proprietary
- [ ] Provide source code access (this GitHub repo)
- [ ] Link to official Linux Mint (https://www.linuxmint.com/)
- [ ] Note any modifications to GPL components
- [ ] Include Arcanus copyright notice

## Approaching Linux Mint Officially

When seeking partnership/licensing from Linux Mint:

1. **Contact:** https://www.linuxmint.com/about.php
2. **Provide:**
   - Link to this GitHub repo
   - Explanation of your use case (demo, internal, etc.)
   - List of components/modifications
   - Branding guidelines for your OS
3. **Discuss:**
   - Trademark usage
   - Distribution methods
   - Support/maintenance plans
   - Revenue sharing (if applicable)

## Legal Risks & Best Practices

### ✅ Dos

- Do attribute Linux Mint and its team
- Do include full GPL license text
- Do maintain source code availability
- Do clearly separate GPL from proprietary components
- Do contact Linux Mint for official partnerships

### ❌ Don'ts

- Don't claim Linux Mint as your own creation
- Don't hide GPL components
- Don't remove GPL notices
- Don't use Arcanus branding to obscure the fact this is Mint-based
- Don't charge money without explicit Linux Mint permission
